# Zephyr LQR Gain Recompute - 2026-05-09

This note records the gain calculation used by the Zephyr Ascento controller.

## Sources

- Zephyr firmware parameters:
  `C:\workplace\wheel_leg\zephyr_ascento_f407_wheel_leg\src\app_config.h`
- Zephyr recompute script:
  `C:\workplace\wheel_leg\zephyr_ascento_f407_wheel_leg\scripts\recompute_lqr_gains.m`
- Reference firmware:
  `C:\workplace\wheel_leg\Venus-1__Four-link-wheel-legged-robot-main`

The Venus firmware was used as a reference for the wheel-leg balance
implementation and sign conventions. Its active balance code is mainly a
cascaded PID implementation, so the LQR values below are recalculated from the
Zephyr physical parameters rather than copied from Venus PID constants.

## Imported Gain Schedule

The MATLAB gain schedule is evaluated from average leg length `L` in metres:

```text
K_pitch      = -59.40269692 * L^2 + 17.93409573 * L - 8.962238054
K_pitch_rate = -54.51745738 * L^2 + 18.40135485 * L - 2.812553242
K_position   =  -3.464101615
K_velocity   = -39.73239567 * L^2 + 14.45755025 * L - 4.034081544
```

The active Zephyr controller now computes the current left/right four-link leg
lengths, averages them, clamps the result to the configured leg-length range,
and evaluates the schedule above every control cycle.

## Stand-Length Check

At the configured stand leg length `L = 0.115 m`, the Zephyr-parameter MATLAB
calculation returned:

```text
K = [-7.6698697090, -1.3976471546, -3.4641016151, -2.8820020530]
```

The fixed `APP_ASCENTO_K_*` defaults in `src/app_config.h` were updated to
these stand-length values as a fallback/reference. Runtime control uses the
scheduled `APP_ASCENTO_GAIN_C*` coefficients.

## Firmware Notes

- `src/app_config.h` contains the recalculated Zephyr-parameter coefficients.
- `src/ascento_balance.c` now uses leg-length scheduled gains instead of fixed
  gains.
- `src/control_loop.c` feeds DM4340 joint feedback into the LQR input before
  evaluating the leg-length schedule.
- `PARAMS_VERSION` was bumped to `19` so old Flash-saved gain parameters do
  not silently override the imported defaults on first boot after flashing.

## Reverification Result

The coefficients were regenerated by running:

```text
cd C:\workplace\wheel_leg\zephyr_ascento_f407_wheel_leg\scripts
result = recompute_lqr_gains();
```

The script reads `src/app_config.h` directly. The model convention is a
per-wheel sagittal equivalent because firmware converts `balance_torque_nm`
directly to each wheel current command. It uses one wheel mass/inertia, one
leg mass, half body mass, and half body pitch inertia. The LQR weights are the
original MATLAB design weights: `Q = diag([1500 1 1200 100])`, `R = 100`.

Wheel output is gated off unless both wheel feedback and both joint feedback
channels are available.
