# Ascento 平衡控制器调参指南

## 1. 系统约束

| 参数 | 值 | 说明 |
|------|------|------|
| 单轮最大电流 | 6000 mA | VESC 限流 |
| 电流→力矩系数 | 0.000180 Nm/mA | KT × 减速比 |
| 单轮最大力矩 | 1.08 Nm | 6000 × 0.000180 |
| 双轮最大力矩 | 2.16 Nm | 1.08 × 2 |
| 平衡角 theta_eq | 0.325 rad (18.6°) | 前倾站立平衡点 |

## 2. 调参历史与教训

### 第一轮：K_pitch=-10, K_pitch_rate=-2.0
- **问题**：12°误差就要 2.1 Nm（超过单轮极限），系统饱和
- **表现**：pitch 在 -12° ~ +57° 振荡，bang-bang 控制
- **教训**：K_pitch=-10 太大，±6° 就饱和

### 第二轮：K_pitch=-5, K_pitch_rate=-1.0
- **改善**：系统大部分时间在线性区，电流 100-1000 mA
- **问题**：平衡点附近恢复力太小，被扰动推离后回不来
- **表现**：可以短暂平衡（17-18°），但被扰动后振荡到 49°
- **教训**：K_pitch=-5 太小，恢复力不足

### 关键发现：K_pitch_rate 在平衡点处"对抗"恢复力

在 t=139.7s 的数据中：
```
pitch=25.51° err=6.89° p_term=0.601 pr_term=-0.258  ← pr_term 阻止恢复
pitch=32.85° err=14.23° p_term=1.242 pr_term=-1.965 ← pr_term 1.6倍于 p_term！
```

当机器人从 25° 向 18.6° 恢复时（pitch_rate < 0），K_pitch_rate=-1.0 产生正力矩，
这正好对抗 p_term 的负恢复力矩。pr_term / p_term 比值在 5° 误差时达到 0.36。

## 3. 力矩预算分析

目标：在 ±10° 误差范围内，总力矩 < 2.16 Nm（双轮极限）。

### 方案 A：K_pitch=-8, K_pitch_rate=-0.5

| 误差 | p_term | pr_term (ω=1 rad/s) | 总力矩 | 饱和？ |
|------|--------|---------------------|--------|--------|
| ±3° (0.052 rad) | 0.42 Nm | 0.50 Nm | 0.92 Nm | 否 |
| ±5° (0.087 rad) | 0.70 Nm | 0.50 Nm | 1.20 Nm | 否 |
| ±10° (0.175 rad) | 1.40 Nm | 0.50 Nm | 1.90 Nm | 否 |
| ±15° (0.262 rad) | 2.09 Nm | 0.50 Nm | 2.59 Nm | 临界 |
| ±20° (0.349 rad) | 2.79 Nm | 0.50 Nm | 3.29 Nm | 是 |

平衡点附近（±5°）的 pr_term/p_term 比值 = 0.71（合理，不再对抗恢复力）。

### 方案 B：K_pitch=-6, K_pitch_rate=-0.5

| 误差 | p_term | pr_term (ω=1 rad/s) | 总力矩 | 饱和？ |
|------|--------|---------------------|--------|--------|
| ±5° | 0.52 Nm | 0.50 Nm | 1.02 Nm | 否 |
| ±10° | 1.05 Nm | 0.50 Nm | 1.55 Nm | 否 |
| ±15° | 1.57 Nm | 0.50 Nm | 2.07 Nm | 否 |
| ±20° | 2.09 Nm | 0.50 Nm | 2.59 Nm | 是 |

更安全但恢复力更弱。

## 4. 推荐参数

**选择方案 A**（K_pitch=-8），理由：
1. 平衡点附近（±5°）恢复力 0.70 Nm，比方案 B 的 0.52 Nm 大 35%
2. ±10° 以内不饱和，给 LQR 足够工作空间
3. ±15° 开始饱和，但此时是大扰动，饱和是可接受的

```
K_pitch        = -8.0   （增加恢复力）
K_pitch_rate   = -0.5   （大幅降低阻尼，避免对抗恢复力）
K_position     = 0.01   （不变）
K_velocity     = 0.5    （不变）
theta_eq       = 0.325  （不变）

pitch_rate LPF τ = 0.050  （3.2Hz，进一步平滑陀螺仪信号）
velocity LPF τ   = 0.020  （8Hz，不变）
```

## 5. 调参路径

如果方案 A 仍然振荡：

| 问题 | 调整 | 理由 |
|------|------|------|
| 平衡点附近恢复力仍不够 | K_pitch → -10 | 增大恢复力矩 |
| 大角度振荡加剧 | K_pitch → -6 | 减小恢复力矩避免饱和 |
| 恢复太快导致冲过头 | K_pitch_rate → -0.8 | 增加阻尼 |
| 阻尼对抗恢复力 | K_pitch_rate → -0.3 | 进一步减小阻尼 |
| 速度信号噪声大 | velocity LPF τ → 0.030 | 加强滤波 |

## 6. 控制律回顾

```
balance_torque = -(K_pitch × pitch_error
                 + K_pitch_rate × pitch_rate
                 + K_position × x_error
                 + K_velocity × v_error)

其中：
  pitch_error = pitch_rad - theta_eq - target_pitch_rad
  符号校正：APP_ASCENTO_IMU_PITCH_SIGN = -1（BMI088 正 pitch → 控制模型负 pitch）
```

## 7. 验证方法

1. 编译烧录
2. `ascento reset` 清除 NVS 旧参数
3. `robot enable 1` 启动平衡
4. 观察日志：
   - 平衡点附近（±5°）：p_term 应 > |pr_term|（恢复力主导）
   - 电流应 < 3000 mA（正常工作范围）
   - pitch 不应超过 ±20°
5. 如果 30 秒内不倒，说明参数可行
