# VESC 电流命令 → 轮端力矩 测量方案

## 目标

测得或修正左右轮各自的 `Nm/mA` 系数，即每 mA 电流命令对应多少 Nm 的轮端力矩。

## 背景

- 轮电机：DJI M3508，减速比 ~19.2:1
- 电调：VESC，CAN 扩展帧协议（CAN2, 1 Mbps）
- 左 VESC ID = 101，右 VESC ID = 100
- 代码中电流命令单位：`current_mA`，经过 `APP_VESC_CURRENT_CMD_TO_AMP = 0.001f` 转换后以安培发送给 VESC
- VESC 反馈的 `current_ma` 是 VESC 实测的电机相电流
- 当前固件默认按 M3508 理论力矩常数和减速比计算左右轮 `torque_k`

关键公式：

```
轮端力矩 τ (Nm) = 拉力 F (N) × 力臂 r (m)
F (N) = 质量读数 m (kg) × 9.81

每 mA 力矩系数 k = τ / I_cmd
将多组数据的 k 取平均或线性拟合斜率，作为左右轮修正值。
```

## 方案选择

### 方案 A：VESC Tool 上位机直连（推荐先做）

VESC 官方上位机（VESC Tool）**可以**用于这项测量，且是最快验证 VESC 电流控制是否正常的方法。

**优点：**
- 不需要 F407 主控参与，排除 CAN 总线问题
- 实时绘图，直观看到电流跟踪
- 可以保存数据日志（Realtime Data → Export）

**缺点：**
- 不能验证 F407 发出的 CAN 命令路径是否与上位机一致
- USB 线缆可能干扰力臂测量（轮子转动受限时电机堵转，力矩最大）

**步骤：**

1. VESC 通过 USB 连接 PC，打开 VESC Tool。
2. 确认 VESC 已检测到电机（运行过 FOC 检测 / 电机向导）。
3. 确认 CAN 相关设置（虽然 USB 模式不需要 CAN，但后续 F407 要用）：
   - Controller ID: 左 101 / 右 100
   - CAN Baud Rate: 1 Mbps
   - CAN Mode: VESC
   - Status Message Rate: 200 Hz
4. 进入 **Manual Control** 页面（或 Realtime Data 页面的控制栏）。
5. 控制模式选择 **Current**（电流控制，非 Duty Cycle 或 RPM）。
6. 输入 1A 电流命令，确认 Realtime Data 中 Motor Current 接近 1A。
7. 输入 -1A，确认电流反向。
8. 逐档测试：0.3A, 0.6A, 1.0A, -0.6A, 2.0A（注意不要超过电机/电调额定值）。
9. 观察 Motor Current 反馈值是否跟踪命令值——如果偏差大，先检查 FOC 检测和电流环 PI 参数。

**物理测量方法（同方案 B）：** 见下方"静态力臂测量法"。

---

### 方案 B：F407 Shell 命令（走生产代码路径，推荐标定时使用）

此方案使用与平衡控制器完全相同的代码路径（`motor_debug_set_wheel_current → dji_m3508_send_current → CAN`），测出的系数直接对应左/右轮各自的 `APP_ASCENTO_*_CURRENT_MA_TO_WHEEL_TORQUE_NM`。

**步骤：**

1. 确认 CAN2 正常，VESC 有反馈：
   ```
   motor can status can2
   motor wheel status all
   ```

2. 用小电流确认方向和堵转安全：
   ```
   motor wheel current left 100 3000
   motor wheel stop
   ```

3. 用机械工装固定力臂（见下方"静态力臂测量法"），逐档发送电流命令：
   ```
   motor wheel current left 300 5000
   ```
   读取拉力计/电子秤读数。

4. 同时用另一个终端读取 VESC 反馈的实际电流：
   ```
   motor wheel status left
   ```
   对比命令电流与 VESC 反馈电流，确认电流跟踪准确。

5. 按公式计算每组的 k 值，取平均。

## 静态力臂测量法

### 所需工具

| 工具 | 用途 |
|------|------|
| 弹簧秤 / 电子秤 / 拉力计 | 测量轮端拉力 |
| 卷尺 / 游标卡尺 | 测量力臂（轮轴到施力点的距离） |
| 固定夹具 / 绳索 | 把拉力施加到固定参考点 |
| 笔记本 + 笔 | 记录数据 |

### 测量步骤

1. **测量力臂 r（轮胎有效半径）**：从轮轴中心到轮胎外缘接地点的距离，单位 m。

2. **固定力臂**（二选一）：
   - **方法 1（推荐）**：在轮子外缘固定一根细绳，绳另一端接弹簧秤，弹簧秤另一端固定在机架或桌面上。电机加电流时，绳拉力 = 弹簧秤读数。
   - **方法 2**：轮子架空，在轮缘切线方向压一个电子秤，通过摩擦力传递力矩。精度较低。

3. **通电测量**：
   - 发送电流命令（例如 `motor wheel current left 300 5000`）。
   - 等待电流稳定（~0.5s），记录弹簧秤/电子秤读数（单位 kg 或 N）。
   - 在超时前（5s 默认最大）记录完成，之后电机自动停止。

4. **逐档重复**：至少 4 个电流档位（含正负），每档测 2-3 次取平均。

5. **数据处理表**：

| I_cmd (mA) | 质量读数 (kg) | F (N) = kg × 9.81 | τ (Nm) = F × r | k (Nm/mA) = τ / I_cmd |
|------------|---------------|---------------------|------------------|------------------------|
| 300        |               |                     |                  |                        |
| 600        |               |                     |                  |                        |
| 1000       |               |                     |                  |                        |
| -600       |               |                     |                  |                        |

6. 对所有有效 k 值取平均，或做线性回归（τ ~ I_cmd 的斜率），得到最终的 `Nm/mA`。

7. 填入 `src/app_config.h`：

```c
#define APP_ASCENTO_LEFT_CURRENT_MA_TO_WHEEL_TORQUE_NM  <左轮测量值>
#define APP_ASCENTO_RIGHT_CURRENT_MA_TO_WHEEL_TORQUE_NM <右轮测量值>
```

## M3508 理论值参考

DJI M3508 电机参数（供对照，最终以实测为准）：

- 扭矩常数 Kt ≈ 0.18 Nm/A（电机轴端）
- 减速比：19.2:1
- 理论轮端扭矩系数：0.18 × 19.2 ≈ **3.46 Nm/A = 0.00346 Nm/mA**

当前代码默认按理论值计算 `torque_k`。如果左右 VESC 上位机校准结果、轮胎、减速器或电流反馈有差异，用左右单独宏覆盖即可。

---

## 注意事项

1. **堵转保护**：VESC 默认有堵转检测，长时间堵转会触发保护。5000ms 的超时足够读数，超时后电机会自动停。
2. **电流限制**：代码中 `APP_M3508_DEBUG_CURRENT_LIMIT` = `APP_WHEEL_CURRENT_SAFE` = 2500 mA，测 1000 mA 以下足够。
3. **方向一致性**：确保正电流 = 正拉力（或负拉力），与 checklist 第 3 节的方向记录对照。
4. **VESC Tool vs F407 电流值**：如果两边测出的 k 值差异 > 5%，优先用 F407 的（因为平衡控制器走的是 F407 → CAN → VESC 这条路径）。
5. **M3508 减速器**：M3508 自带行星减速器（~19.2:1），测量的是减速后的轮端力矩，不是电机轴力矩。
6. **左右分开测**：左右两轮可能因为减速器个体差异有 2-5% 的差异，建议分别标定。
